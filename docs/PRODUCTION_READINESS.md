## 🚧 BLOCKERS — things only Charlie can fix

### 1. DNC scrubbing — REALPHONEVALIDATION_API_KEY missing (CRITICAL)

**What's broken**: All cold call/SMS dispatch is BLOCKED with `fail-safe-deny` because no DNC provider is configured. This is intentional — sending unsanitized cold messages risks $500–$1,500 TCPA violations per number.

**Impact**: Drip campaigns can email but cannot SMS, voicemail, or call cold leads.

**Fix**: Sign up at [realphonevalidation.com](https://realphonevalidation.com), they email an API key (typically same-day to 24h). Send the key to me as `REALPHONEVALIDATION_API_KEY=...` and I push to Vercel. DNC scrubbing turns on instantly + drip engine starts dispatching SMS.

**Alternative**: If RealPhoneValidation is too slow, get your own federal SAN at telemarketing.donotcall.gov (~$100/yr) and we use a different provider. But RPV is faster and covers state DNCs + TCPA litigators which a federal SAN alone doesn't.

### 2. Firecrawl — out of credits (LOW URGENCY)

**What's broken**: Hailey's `firecrawl_scrape` tool returns `insufficient_credits`. Key auths fine — just needs quota.

**Impact**: Hailey can't scrape JS-heavy pages (Reddit, NextDoor, county records that block normal fetch). Tavily + Exa + Jina still work for most research. BrowserUse handles the rare cases that need actual browser interaction.

**Fix**: At [firecrawl.dev/pricing](https://firecrawl.dev/pricing), add a payment method or buy a credit pack. Smallest pay-as-you-go pack is fine for testing. No env var change needed — same key just starts working.

**Alternative**: Skip Firecrawl entirely. Hailey already has tavily_search + exa_general_search + exa_social_search + jina_read_url + browseruse_run. Firecrawl is the "nice to have" 6th option.

### 3. Google Drive uploads — service account quota issue (MEDIUM URGENCY)

**What's broken**: Drive READ works (listFiles returns files). Drive UPLOAD fails with "Service Accounts do not have storage quota."

**Impact**: `parse_estimate` saves the parsed JSON to the documents table + appends line items to the Sheet, but the original PDF doesn't get backed up to Drive. Charlie loses the file copy.

**Fix (Path A — recommended)**: Create a Google Workspace **Shared Drive**:
1. Open `drive.google.com`
2. Click **+ New → Shared Drive** (or **My Drive → Shared Drives**)
3. Name it "Just Hail Hailey"
4. **Move the existing CMBF 2026-2027 folder INTO the Shared Drive** (or create a new one inside)
5. Right-click the Shared Drive → Manage Members → add `ga-reader@just-hail-website.iam.gserviceaccount.com` as Content Manager (Editor)
6. Get the new folder's ID from URL → send to me as the new `GOOGLE_DRIVE_ROOT_FOLDER_ID`

**Fix (Path B — accept the loss)**: Estimate parsing still works without Drive backup. The parsed structured data lands in `documents` table + Sheets ledger. Just no PDF copy stored.
